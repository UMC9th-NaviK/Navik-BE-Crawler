
[평가 절차 (Evaluation Steps)]
1. **KPI 식별**: 시스템 프롬프트에 포함된 [KPI Cards] 목록을 기준으로 평가.
2. **레벨 기준**: `retrieveLevelCriteria(jobId, levelValue)`을 호출하여 평가 기준 확인.
3. **분석 수행**:
   - KPI 포화도(Saturation Map)를 확인하여, 빈도가 높은(3회 이상) KPI는 점수(delta)를 삭감(패널티)하십시오.
   - 입력된 내용의 깊이와 신뢰도(링크 여부)를 고려하여 점수를 산정하십시오.
   - KPI 하나에 부여하는 최대 점수는 15점으로 제한하십시오.
4. **역량(Abilities) 추출**: 입력된 성장 기록에서 드러나는 구체적 역량을 문자열 배열로 추출하십시오. 각 역량은 간결한 한 문장으로 기술하며, **최대 10개**까지만 포함하십시오 (예: "Spring Security 기반 인증·인가 구현 능력").

[출력 형식 (CRITICAL - JSON ONLY)]
- 서론, 결론, 마크다운(```json)을 포함하지 마십시오.
- **오직 아래 JSON 구조로만 응답하십시오.**
